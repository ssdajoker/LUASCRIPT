Arrow Functions

Arrow functions provide a concise syntax for writing functions in LUASCRIPT. They support both expression and block bodies, parameter destructuring, and maintain lexical scope for closures.

Basic Syntax

// Single parameter (parentheses optional)
let double = x => x * 2;

// Multiple parameters (parentheses required)
let add = (a, b) => a + b;

// No parameters
let greet = () => "Hello, World!";

// Block body with explicit return
let multiply = (x, y) => {
    let result = x * y;
    return result;
};

Advanced Features

Closures and Higher-Order Functions

// Closure example
let makeAdder = x => y => x + y;
let add5 = makeAdder(5);
print(add5(3)); // Output: 8

// Array-like operations (conceptual)
let numbers = [1, 2, 3, 4, 5];
let doubled = numbers.map(x => x * 2);
let filtered = numbers.filter(x => x > 2);

Memory Management

LUASCRIPT Phase 1D introduces comprehensive memory management to ensure reliable execution and prevent common issues like memory leaks and stack overflows.

Parser Memory Management

The parser includes built-in memory limits to prevent excessive resource usage during compilation:

// Initialize parser with memory limits
const memoryManager = new MemoryManager(
    10000,  // Maximum AST nodes
    100     // Maximum recursion depth
);

const parser = new Parser(tokens, memoryManager);

// Memory stats are available during parsing
const stats = memoryManager.getStats();
console.log(`Nodes: ${stats.nodeCount}/${stats.maxNodes}`);
console.log(`Depth: ${stats.currentDepth}/${stats.maxDepth}`);

Runtime Memory Management

The runtime environment provides heap management and call stack protection:

// Runtime memory manager with limits
const runtimeMemory = new RuntimeMemoryManager(
    1000,           // Maximum call stack depth
    50 * 1024 * 1024 // Maximum heap size (50MB)
);

// Automatic garbage collection triggers
// Memory stats available during execution
const interpreter = new Interpreter();
const stats = interpreter.getMemoryStats();
console.log(`Heap usage: ${stats.heapUtilization}`);

Memory Safety Features

  • Stack Overflow Protection: Prevents infinite recursion
  • Heap Limit Enforcement: Prevents memory exhaustion
  • Automatic Garbage Collection: Frees unused memory
  • Resource Cleanup: Proper cleanup on errors
  • Memory Statistics: Real-time monitoring
Performance Note: Memory management adds a small overhead but significantly improves reliability. Limits can be adjusted based on your application's requirements.

Parser Features

The Phase 1D parser includes enhanced error handling, memory management, and support for modern syntax features.

Enhanced Error Reporting

// Detailed error messages with location information
try {
    const ast = parser.parse();

// Parser automatically manages memory allocation

Code Examples

Complete Arrow Function Examples

// Basic arrow functions
let square = x => x * x;
let add = (a, b) => a + b;
let greet = name => `Hello, ${name}!`;

// Arrow functions with block bodies
let factorial = n => {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
};

// Higher-order functions
let compose = (f, g) => x => f(g(x));
let addThenSquare = compose(square, x => x + 1);

print(addThenSquare(4)); // Output: 25 (4+1=5, 5*5=25)

// Closures
let counter = () => {
    let count = 0;
    return () => ++count;
};

let increment = counter();
print(increment()); // 1
print(increment()); // 2

Memory Management Example

// Example with memory monitoring
let fibonacci = n => {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
};

// This will trigger memory management
try {
    let result = fibonacci(40);
    print(`Fibonacci(40) = ${result}`);
} catch (error) {
    if (error.message.includes('Stack overflow')) {
        print('Recursion too deep - consider iterative approach');
    }
}

API Reference

MemoryManager Class

class MemoryManager {
    constructor(maxNodes = 10000, maxDepth = 100)
    allocateNode(type, data = {})
    enterScope()
    exitScope()
    cleanup()
    getStats()
}

RuntimeMemoryManager Class

class RuntimeMemoryManager {
    constructor(maxCallStack = 1000, maxHeapSize = 50MB)
    enterFunction(functionName, args = [])
    exitFunction()
    allocateObject(obj, estimatedSize = 64)
    triggerGarbageCollection()
    getStats()
    cleanup()
}
